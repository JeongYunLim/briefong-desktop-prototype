# 브리퐁(Briefong) 데스크톱 프로토타입

![브리퐁 대표 이미지](./briefong-icon.png)

> **브리핑을 필요한 것만 퐁당퐁당**<br>
> 복잡한 이커머스 페이지에서 사용자가 원하는 핵심 정보에 더 빠르게 접근하도록 돕는 데스크톱 접근성 서비스 프로토타입입니다.

## 프로젝트 소개

기존 스크린리더는 웹페이지의 DOM 순서를 따라 콘텐츠를 선형적으로 읽습니다. 이커머스 페이지처럼 광고, 추천 상품, 메뉴, 배너가 많은 환경에서는 상품명이나 가격, 배송 정보, 구매 버튼에 도달하기까지 불필요한 내용을 오래 들어야 할 수 있습니다.

브리퐁은 이러한 탐색 부담을 줄이기 위해 페이지의 핵심 정보를 선별하고, 사용자가 원하는 정보로 바로 이동할 수 있는 **비선형 접근성 탐색 경험**을 제안합니다.

이 저장소는 실제 쿠팡 페이지나 브라우저 확장 프로그램과 연결된 완성 제품이 아니라, 브리퐁의 사용자 흐름과 기대 효과를 설명하기 위한 **교육·연구 목적의 인터랙티브 웹 프로토타입**입니다.

## 문제와 제안

| 기존 탐색(Before) | 브리퐁 적용(After) |
| --- | --- |
| 페이지 상단부터 순서대로 낭독 | 핵심 정보 중심으로 재구성 |
| 광고·메뉴·추천 영역까지 탐색 | 상품명·가격·배송·리뷰·구매 영역 우선 제공 |
| 원하는 정보까지 반복적인 키 조작 | 음성 명령 또는 단축키로 직접 이동 |
| 정보의 위치와 전체 구조 파악이 어려움 | 현재 정보 역할과 다음 탐색 위치를 명확히 안내 |

## 주요 기능

### 1. Before → After 탐색 비교

- 일반적인 선형 낭독 과정과 브리퐁의 핵심 정보 우선 탐색을 애니메이션으로 비교합니다.
- 같은 목표 정보에 도달하는 과정이 어떻게 달라지는지 시각적으로 확인할 수 있습니다.

### 2. 음성 상품 검색 시연

- `Shift + Space`로 음성 상품 검색 흐름을 시작합니다.
- 프로토타입에서는 “무선 노이즈 캔슬링 헤드폰”이라는 예시 입력이 자동으로 표시됩니다.
- 실제 음성 인식 API가 아닌 사용자 경험 시연용 동작입니다.

### 3. 검색 결과와 상세 페이지 이동

- 예시 검색 결과 3개를 탐색합니다.
- 첫 번째 상품을 선택하거나 음성 명령 흐름을 통해 상세 페이지로 이동합니다.
- 상품명, 가격, 평점, 배송과 같은 가상 정보를 제공합니다.

### 4. 핵심 정보 중심 접근성 뷰

- 상품 상세 정보를 `상품명 → 가격 → 배송 → 리뷰 → 구매` 순서로 재구성합니다.
- 상단 버튼, 우측 탐색점, 스크롤 또는 단축키로 원하는 영역에 이동할 수 있습니다.
- 각 영역의 “읽어주기” 버튼은 브라우저의 Web Speech API를 이용해 한국어 음성을 출력합니다.

### 5. 키보드 단축키

| 단축키 | 동작 |
| --- | --- |
| `Shift + Space` | 음성 상품 검색 시연 시작 |
| `Space` | 음성 명령 시연 시작 |
| `Alt + Z` | 접근성 뷰 켜기/끄기 |
| `Alt + P` | 상품명 영역으로 이동 |
| `Alt + C` | 가격 영역으로 이동 |
| `Alt + D` | 배송 정보 영역으로 이동 |
| `Alt + R` | 리뷰 영역으로 이동 및 안내 재생 시연 |
| `Alt + B` | 구매 버튼 영역으로 이동 |

입력창, 텍스트 영역 또는 편집 가능한 요소에 포커스가 있을 때는 페이지 단축키가 작동하지 않도록 구성했습니다. 운영체제나 브라우저의 기본 단축키와 충돌하는 경우 화면 버튼으로 동일한 흐름을 체험할 수 있습니다.

## 체험 순서

1. 페이지 상단의 **같은 목표로 비교 재생**을 눌러 Before와 After를 비교합니다.
2. **프로토타입 직접 체험하기**를 누르거나 아래 데스크톱 화면으로 이동합니다.
3. 좌측의 **전체 과정 자동 시연**을 누르면 6단계 흐름이 자동 재생됩니다.
4. 직접 조작하려면 단계별 메뉴, 화면 버튼 또는 단축키를 사용합니다.
5. 접근성 뷰에서 각 정보 영역의 **읽어주기** 버튼을 눌러 한국어 TTS를 확인합니다.

## 실행 방법

별도의 설치나 빌드 과정이 필요하지 않습니다.

### 가장 간단한 방법

저장소의 `index.html`을 더블클릭하여 최신 Chrome 또는 Edge에서 엽니다.

### 로컬 서버로 실행

```bash
python -m http.server 8000
```

브라우저에서 `http://localhost:8000`에 접속합니다.

## 파일 구조

```text
.
├── .github/
│   └── workflows/
│       └── pages.yml       # GitHub Pages 자동 배포
├── .nojekyll               # Jekyll 처리 없이 정적 파일 제공
├── briefong-icon.png       # 브리퐁 대표 이미지와 화면 아이콘
├── index.html              # HTML, CSS, JavaScript가 포함된 프로토타입
└── README.md               # 프로젝트 설명
```

## 사용 기술

- HTML5
- CSS3
- Vanilla JavaScript
- Web Speech API (`SpeechSynthesisUtterance`)
- GitHub Actions / GitHub Pages

외부 프레임워크, 패키지, 서버 또는 데이터베이스는 사용하지 않습니다.

## 배포

`main` 브랜치에 변경 사항을 푸시하면 `.github/workflows/pages.yml`이 정적 파일을 GitHub Pages에 자동 배포합니다.

최초 한 번은 저장소의 **Settings → Pages → Build and deployment → Source**에서 **GitHub Actions**를 선택해야 할 수 있습니다. 배포 진행 상태는 저장소의 **Actions** 탭에서 확인할 수 있습니다.

## 프로토타입의 범위와 제한

- 데스크톱/노트북 화면을 기준으로 설계되어 최소 화면 너비가 큽니다.
- 실제 쿠팡 상품 검색, 회원 기능, 구매, 결제와 연결되지 않습니다.
- 화면의 상품명, 가격, 평점, 리뷰, 배송 정보는 모두 시연용 가상 데이터입니다.
- 음성 검색과 음성 명령은 실제 음성 인식이 아니라 미리 설계된 시나리오를 보여줍니다.
- 읽어주기 기능은 브라우저와 운영체제가 제공하는 음성 합성 기능 및 한국어 음성 설치 여부에 따라 다르게 동작할 수 있습니다.
- 실제 접근성 효과를 입증하려면 시각장애인 사용자 평가, 스크린리더별 호환성 검증, 키 충돌 테스트와 정량 지표 측정이 추가로 필요합니다.

## 연구·교육 목적 안내

본 사이트는 교육·연구 목적의 비공식 서비스 프로토타입입니다. 쿠팡 및 관련 기업과 제휴·승인 관계가 없으며, 실제 상품 검색·구매·결제 기능을 제공하지 않습니다. 화면의 상품명·가격·리뷰는 시연을 위한 가상 정보입니다.

## 향후 발전 방향

- Chrome Extension의 DOM 파싱·중요도 계산·가상 DOM 생성 파이프라인과 연동
- 동적으로 추가되는 상품, 광고, 모달을 처리하는 `MutationObserver` 기반 갱신
- 실제 음성 인식과 자연어 명령 처리
- NVDA, JAWS, VoiceOver 등 스크린리더별 사용성 검증
- 단축키 사용자 설정과 충돌 감지
- 사용자 실험을 통한 탐색 시간, 키 입력 수, 과업 성공률 비교

## 팀 프로젝트

브리퐁은 시각장애인의 이커머스 정보 접근 경험을 개선하기 위해 진행 중인 **Re-Creator Crew(리크크)** 팀 프로젝트의 서비스 프로토타입입니다.
